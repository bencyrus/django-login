  1. The application utilizes Django's default User model with modifications to enhance functionality within the admin panel.
  2. The configuration for static and media URLs is defined in the settings.py file and references the static folder for resources.
  3. A custom user creation form is implemented to create new users, which overrides the default form provided by Django.
  4. Access to the home page is restricted to authenticated users only.
  5. If a user is not authenticated, they will be redirected to the login page for authentication.
  6. Once a user is authenticated, they will be redirected to the home page.
  7. Upon logging out, the user will be redirected to the login page.
  8. After registering, the user will be redirected to the home page.
  9. The database currently in use is SQLite3.
  10. JWT for tokens and authorization or another database like PostgreSQL could be used to store user data given more time.
  11. Validation is implemented for the registration form on the front end to ensure that the user enters a valid username, email, and password.
  12. Login validation is handled by the backend and error messages are displayed to the user if the login credentials are invalid.
